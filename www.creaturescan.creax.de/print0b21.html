cS_engine mit nächster Version

ab kommender Version wird es die Möglichkeit geben, neben der "Netz-Synchronisation" auch mit "php-Sync" die Daten übers Netz zu tauschen.

Die alte Lösung, die mehr auf peer to peer basiert bzw. einen windows server vorrausgesetzt hat, ist wohl dochnicht die optimale lösung wie ich mir das so vorgestellt habe. Zwar funktioniert das ganze inzwischen doch ganz gut, aber leider ist ein Server mit creatureScan nicht sehr stabil (wie ich selbst festgestellt habe). Und vielen Nutzern ist es auch garnicht möglich direkt den Port ins Internet durchzuschalten.

Darum wird ab der neuen Version eine Funktion bereitgestellt die es ermöglicht die Daten über einen "normalen" Webspace mit MySQL (wie bei vielen freehostern umsonst verfügbar) auszutauschen.

Ich habe dazu auchschon einen Eintrag im Wiki erstellt:
http://www.creaturescan.creax.de/wiki/index.php5/CS_engine

Zur Konfiguration des Clients(creatureScan) mach ich auchnoch einen Eintrag.

ich spiele auch mit dem Gedanken eine "öffentliche" Datenbank für einige Unis anzubieten. Allerdings ist hier die Gefahr der Dateninkonsistenz recht hoch....
