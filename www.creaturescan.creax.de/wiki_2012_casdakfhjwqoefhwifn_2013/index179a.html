CS engine

Aus creatureScan
Wechseln zu: Navigation, Suche

Die cS_engine ist ein Stück php-Software welches creatureScan ermöglicht Scanberichte und Sonnensysteme über eine MySQL Datenbank auszutauschen.

Da diese Bedienungsanleitung vorallem für unerfahrene webadmins sehr kurzgefasst ist, gibt es hier noch eine Schritt für Schritt-Anleitung:

Schritt für Schritt How-To: HowTo_cS_engine

Systemvorausetzungen

PHP, mit XML Unterstützung. MySQL Datenbank.

Einrichtung

download php

Lade dir unter dieser Adresse den aktuellsten php-Code runter:

http://www.creaturescan.creax.de/cS_engine/svn/

entpacke das Zip-Archiv.

Lösche folgende Dateien (werden nicht benötigt): 

 sxml_test.php
 cSmod.php

Übertrage die Dateien (z.B. per ftp) auf deinen Webspace, am besten in einen eigenen Unterordner (z.b.: "cS_engine")

config.php anpassen

Bearbeite die Datei "config.php" nach deinen Bedürfnissen. Trage dabei mindestens folgende Werte ein: 

 $universe = '<uniname>.<ogame-domain>';        // bsp: andromeda.ogame.de uni17.ogame.de
 $sqlconf['mysql_server'] = 'localhost';        // passt normalerweise so
 $sqlconf['mysql_username'] = '<login für mysql>';
 $sqlconf['mysql_password'] = '<passwort>';
 $sqlconf['mysql_db'] = '<datenbankname>';      // oft gleich wie mysql_username

Führe alle SQL Befehle in der Datei "db_structure.sql" auf deinem MySQL Server aus (z.B. mit phpMyAdmin). Dabei werden alle benötigten Tabellen für die cS_engine angelegt.

Benutzer einrichten

Die Benutzer werden in der MySQL Datenbank gespeichert. Die Tabelle "users" ist dafür vorgesehen. Da die Web-Oberfläche noch nicht soweit ist, musst du die User mit Hilfe von phpMyAdmin eintragen:

Für das Passwortfeld musst du die Funktion MD5 einstellen, da das Passwort nur als MD5 Checksumme gespeichert wird.

Serverseitige Scripte Updaten

Wenn es eine neue Version der cS_engine Scripte gibt, reicht es nicht immer aus die alten Scripte einfach zu ersetzen. Bei manchen Versionen gibt es auch Änderungen an den Tabellenstrukturen in der Datenbank.

rev15 auf rev 180: siehe Post in Forum: http://www.creaturescan.creax.de/thread.php?postid=769#post769
