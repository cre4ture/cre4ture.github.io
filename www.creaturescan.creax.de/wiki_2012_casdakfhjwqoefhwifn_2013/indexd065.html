Diese Seite ist veraltet. Diese Funktionalität wird nicht weiter von cS unterstützt.


Mit Hilfe dieser Funktion ist es möglich sämtliche Daten, die in cS eingelesen wurden (Spios, Sys, Stats, Flotten) direkt mit einem andern cS-user zu synchronisieren. (peer to peer)

Solange die Verbindung zwischen Zwei cS-Programmen nicht unterbrochen wird, werden alle Daten synchron gehalten, d.h. wenn der eine einen Scan einließt, bekommen ihn alle anderen, mit denen eine Verbindung besteht, auch diesen Scan, sofort!

[Bearbeiten] Server Konfiguration

Hier kann auch DynDNS benutzt werden! Hier kannst du, als Server z.B. deine aktuelle IP nachschauen: http://www.creax.de/myip.php Der Server muss an alle Clients, die verbinden wollen, die IP verteilen (übers Forum, email, msn, icq,......)

Bildbeschreibung:

- Beim Server müssen zu erst Benutzer über die Benutzerverwaltung angelegt werden. Diese Logindaten müssen dann die Clients verwenden um sich erfolgreich mit dem Server zu verbinden.

- Um den Server zu starten, muss der Haken bei "Server aktiv" gesetzt sein (rot markiert). Wenn "starte Server bei Programmstart" aktiviert ist, wird dieser Haken automatisch nach dem start von creatureScan gesetzt.

[Bearbeiten] Client Konfiguration

Anstelle von "localhost" sollte an dieser Stelle natürlich die öffentliche IP-Addresse des Servers verwendet werden! Hier kann auch DynDNS benutzt werden! Hier kannst du, als als Server z.B. deine aktuelle IP nachschauen: http://www.creax.de/myip.php

WICHTIG: DU MUSST DIE IP DES SERVERS EINTRAGEN!!! UND NICHT DEINE EIGENE!!! Der Spieler, der sein cS als Server konfiguriert hat, muss allen Clients seine IP zukommen lassen!!!

Bildbeschreibung:

- Beim Client müssen vor der Verbindung auch Rechte vergeben werden, somit kann man auch von der Clientseite den Zugriff des Servers beschränken. Im Normalfall werden aber einfach alle Haken bei "Rechte der Verbindung zum Server" aktiviert.

- Danach kann der Client sich mit "Verbinden" zum Server verbinden. Hier muss dann ein Benutzer und zugehöriges Passwort eingetragen werden. Diese Informationen musst du vom Server holen.
