ein workaround fuer foxgame waere dieser...

in der foxgame.jar befindet sich die datei foxgame.js. dort koennte man eine zusaetzliche funktion (fg_meineFunktion) einfügen, z.b. an folgender stelle:

code:
//***********************************************************************
// MAIN function
//***********************************************************************
...
   fg_addReportButtons,
   fg_addMilosButtons,
   fg_addRankingButtons,
   fg_meineFunktion,
   fg_addDebrisImage,
   fg_removePlanets,
...

die eigentliche funktion platziert man weiter unten im source zwische beliebigen anderen funktionen.

fg_meineFunktion koennte so aussehen:

code:
//***********************************************************************
function fg_meineFunktion(document) {

   if (document && !fg_isGalaxyUrl(document.location.href)) return;
   if (fg_Debug) alert("fg_meineFunktion");

   try {
      goDoCommand('cmd_selectAll');
      goDoCommand('cmd_copy');
      goDoCommand('cmd_selectNone');
   } catch (e) {
      foxgamedebug(e);
   }
}

folgendes wuerde passieren: sobald man eine gala-ansicht in unserem lieblingsspiel offen hat, wird automatisch die komplette seite markiert, in die zwischenablage kopiert und die markerung wieder entfernt. man bekommt vom kopieren in die zwischenablage also nicht ein mal etwas mit.

hmm, tja, alles theorie...
